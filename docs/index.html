Why zgx

  • One binary, every desktop. Pure-Go, no cgo — cross-compiles to Linux/macOS/Windows (amd64 + arm64).
  • Zero-config discovery. Finds devices on the LAN via mDNS.
  • Safe SSH bootstrap. ed25519 keys, trust-on-first-use host-key confirmation, idempotent setup.
  • Curated AI/ML stack. 17 apps installed in dependency order with partial-failure reporting.
  • Transparent installs. Shows raw remote commands and flags curl | sh before running.
  • High-speed pairing. Discovers ConnectX NICs and writes link-local netplan.

Install

Pick whichever is easiest:

# macOS / Linux — Homebrew (auto-updates):
brew install --cask AndersSol/tap/zgx

# macOS — double-click installer: download the .pkg above
# (signed + notarized — no Gatekeeper warning), double-click, done.

# Any OS — with Go:
go install github.com/AndersSol/zgx-cli@latest

# …or build from source (Go 1.26+):
git clone https://github.com/AndersSol/zgx-cli.git && cd zgx-cli && go build -o zgx .

Quick start

zgx discover                              # find devices on the network
zgx connect zgx-ab12cd --user hp          # set up SSH key access
zgx list                                  # browse the app catalog
zgx install jupyter-lab --host zgx-ab12cd # install (deps pulled in automatically)
zgx health zgx-ab12cd                      # verify connectivity

Commands

CommandWhat it does
discoverFind ZGX devices on the network via mDNS.
connected25519 key setup + SSH alias + key-access verification.
listPrint the curated app catalog.
installInstall apps over SSH in dependency order.
verifyCheck whether apps are installed.
uninstallRemove named apps (shared deps kept).
healthVerify SSH connectivity.
dns-registerRegister with Avahi for stable mDNS rediscovery.
pair / unpair / pair-detailsConfigure / remove / inspect ConnectX pairing.
configSave and manage known devices.

Security

  • Host-key TOFU + fingerprint. Confirms an unknown host's fingerprint before the password is sent; rejects changed keys.
  • No surprise code. Raw remote commands shown; download-and-execute lines require explicit confirmation.
  • Input validation. No SSH-config injection; NIC names and device IDs validated before use in remote commands.
  • No secrets. Passwords never logged or written to disk; repo is gitleaks-clean.
